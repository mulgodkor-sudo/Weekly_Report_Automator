# 패치 안내 — apply_patch_v1.11.bat

> 기준선(baseline): **v1.1** (최초 README 작성 시점)
> 패치 후 버전: **v1.11**

---

## 사용 방법

1. `Weekly_Report_Automator_VX.X.exe` 가 있는 폴더(= `src/` 폴더가 있는 폴더)에
   `apply_patch_v1.11.bat` 파일을 복사한다.
2. `apply_patch_v1.11.bat` 더블클릭 실행.
   - `src/` 폴더가 없으면 즉시 에러로 중단 (잘못된 위치에서 실행한 경우).
   - 패치 데이터는 base64 → zip으로 디코딩 후 `src/` 폴더에 **전체 파일 교체** 방식으로 적용된다 (부분 수정 없음).
   - 적용 중 생성되는 임시 파일(`%TEMP%\wra_patch_*.b64`, `*.zip`)은 성공/실패와 관계없이
     스크립트 종료 시 자동 삭제된다. **백업 폴더는 생성하지 않는다.**
   - 이어서 `.exe` 파일명과 그 `.exe`가 들어있는 폴더명에 들어있는 "V1.1"을
     "V1.11"로 자동 변경한다 (예: `Weekly_Report_Automator_V1.1.exe` →
     `Weekly_Report_Automator_V1.11.exe`, 폴더 `...V1.1\` → `...V1.11\`).
     이름에 "V1.1" 패턴이 없거나 이미 "V1.11"로 되어 있으면 건너뛴다(경고만 출력).
3. 패치 완료 후 프로그램을 재시작하면 적용된다. (폴더명이 바뀌었으므로 바로가기를
   쓰고 있다면 새 경로로 다시 만들어야 한다)

> ⚠️ `main.py`는 PyInstaller가 `.exe`에 직접 빌드해 넣는 진입점이라 이 패치로
> 갱신할 수 없다. main.py가 바뀐 경우는 `build.bat`로 다시 빌드해서 `.exe` 자체를
> 교체해야 한다. (이번 v1.11 패치에는 main.py 변경 내용이 없으므로 해당 없음)

## exe / 폴더명 변경 관련 주의사항

- exe 파일과 그 exe가 들어있는 폴더의 rename은 패치 핵심 동작(파일 교체)이
  끝난 **이후**에 실행되므로, 이름 변경이 실패해도 `src/` 패치 자체는 이미
  정상 적용된 상태다.
- rename이 실패하는 경우(프로그램이 실행 중이거나, 해당 폴더를 탐색기/cmd 창이
  열어두고 있는 경우 등)는 경고 메시지로 새 이름을 안내하니, 안내된 이름으로
  수동으로 바꿔주면 된다.
- 폴더 자체의 이름을 바꾸는 동작은(`apply_patch_v1.11.bat`가 실행되고 있는
  바로 그 폴더) Windows에서 일반적으로 잘 동작하는 방식(부모 폴더로 이동 후
  rename)으로 구현했지만, 실제 환경에서의 동작은 팀 배포 전 1회 직접
  확인이 필요하다.

## 검증 상태

- base64 → zip 디코딩 결과는 원본 `src/*.py`, `src/*.json` 파일과 바이트 단위로 동일함을
  개발 환경에서 직접 확인했다.
- 실제 팀원 PC(Windows + PowerShell)에서 `src/` 패치 적용은 1회 실행 테스트 완료, 정상 동작 확인.
- exe/폴더 rename 기능은 이번에 새로 추가된 부분으로, 아직 실제 Windows 환경에서
  테스트되지 않았다. 배포 전 반드시 한 번 직접 실행해서 확인할 것을 권장한다.
